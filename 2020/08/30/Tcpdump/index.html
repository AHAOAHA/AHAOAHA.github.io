Tcpdump
發表於 2020-08-30| 更新於 2020-08-30
閱讀量:

tcpdump学习记录。
tcpdump是一款用来抓包的工具，通常使用在Linux环境中。使用tcpdump必须具有当前机器的root权限。

最简单的使用

sudo tcpdump -i eth0: 监听eth0网卡上的所有数据包，这个命令的功能与sudo tcpdump是一样的，如果tcpdump没有添加任何其他参数，会默认监听第一个网卡上的数据包，结果会以IP包为单位输出。

tcpdump可以识别的协议

  • ARP
  • RARP
  • ICMP
  • TCP
  • UDP
  • IP
  • IPv6
  • AppleTalk
  • IPX
  • SSH

参数1 -nn

-nn参数用来禁止tcpdump内部默认的协议转换，例子如下：
当使用sudo tcpdump时，由22端口抓到的包结果输出如下：22:45:07.318782 IP 120.245.130.10.8558 > localhost.ssh: Flags [P.], seq 37:73, ack 58344, win 514, length 36
当使用sudo tcpdump时，由22端口抓到的包结果输出如下：22:47:26.189674 IP 120.245.130.10.8558 > 172.16.24.93.22: Flags [P.], seq 37:73, ack 63096, win 514, length 36

参数2 -i

-i通过该选项来指定要监听的网卡，网卡需要通过ifconfig查询

参数3 -X

如果不添加该参数，输出的结果只有包的流向以及长度等信息，通过该选项可以输出每个抓到的包的具体内容，协议头以及包内部的数据信息，输出的包内数据会以16进制以及ASCII的形式显示。

参数4 -t

-t隐藏输出内容中的时间戳信息

参数5 -c

-c指定需要抓包的个数，当抓到指定个包后即退出

参数6 -s

tcpdump抓包时默认每个包的大小为68bytes，当包的大小超出68bytes时间就会对包的内容进行截断，使用-s可以指定该截断包的阈值，当使用-s 0时表示抓取全量的包数据。

参考

sudo tcpdump -i eth2 -A -s 0 -vv ' tcp port 8889 and (((ip[2:2] - ((ip[0]&0xf)<<2)) - ((tcp[12]&0xf0)>>2)) != 0)'

  • ip[2:2]：取出IP协议头中的数据总长度

  • (ip[0]&0xf)<<2

  • tcp[12]&0xf0)>>2

抓取eth2网卡上流过8889端口的数据包，过滤条件：IP的有效数据包减去IP头部和TCP头部大小之后不为空，即有效数据不为空
